Introduction

I am currently making games mainly as a hobby. It's fun and fills time. I also improve my general coding skill by doing so.
I was inspired to make games when i got curious about how other games worked, so I tried to make my own. I started as a noob following tutorials and I'm still learning a lot about game design and structure.
In the future, especially when I'll be 18 I might consider making games for a bit of money.

Tools

Pygame: the first language I learned was python, so obviously for my first game attempts I used the clearly best game library for python: Pygame.
Even if it has no user interface or complex functionality built-in like a game engine I still enjoy it a lot thanks to the control it gives me over my code and thanks to the dynamic style of python.
Recently I also switched to pygame-ce, the clearly better version of the two.

Unity Engine: after python I got to the C# language, my personal favourite so I started using Unity.
the differences against pygame are immense, and it's technically "better" but it's actually preference based on the user needs. One is a game engine, the other is a game framework. Personally I like them both, mainly thanks to their differences.
Also Unity is best know for its 3D while pygame for 2D, so they complement each other.
