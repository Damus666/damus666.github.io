Overview

I said i'm also a programmer: even tho games are mainly code, I don't make just games. Sometimes i need to take a break or just have an inspiration that is not a game. My favourite languages are Python and C#. I made only one real project with C#, i solely used it for games, instead i've made plenty of projects with python, probably because it's easier and more accessible. Most of them still rotate around the pygame library somehow, but there are a few that don't.

You can find all the projects I'll be listing and more in my github repositories.

Programming Languages

  • Damuscript: I finally done what I've always wished to do: a feature complete programming language. It's interpreted in python and has an API to add features to it. Not the best in terms of performance, but at least i managed to make it.
  • Solace: Damuscript predecessor, a way less complex programming language. The base is from a tutorial, that's why i wasn't able to expand it as i would have liked. Damuscript solves all its issues.

Libraries

  • PGPU (Pygame GPU Accelerated Engine): The library I'm more proud of is for sure PGPU. It's a engine wrapper for the pygame _sdl2 module, still in developement, which let you use your GPU. I was inspired by the Unity Component System for the engine structure.
  • DamGUI: A pygame library containing various UI components. The structure is inspired by ImGUI, that's why i like it a lot. The code is super messy tho, so I might rewrite the whole library.
  • pygameUI: An older UI library than DamGUI. I don't think it's inferior tho, as it has a completely different structure. In fact, it uses python object to describe UI components, instead of runtime function calling. Might even be faster, but it depends on the user preference to use one or another.
  • PGBoost and pygame_helper: Both are pygame libraries that should help developing games. They are not great tho imo, I think PGPU is better under any point of view.
  • C# Engine: An attempt of a game engine in C# using SDL2, inspired by pygame. It's my only uploaded C# project, and i think PGPU is better than this.
  • Json Helper: A python module that converts JSON strings into a set of python objects, to manage the JSON easly. It also contains a bunch of utility functions and classes. Pretty neat.

Tools

  • Tile Kit: A not so good but decent level editor featuring tiles and objects plus some tools and saving/loading. I could make a better one, for now i got this tho, and it's powered by pygameUI.
  • Graph Wizard: A graphic calculator able to render any function into a XY plane. It's powered by pygame and pygameUI, Performance can drop easly. It also support variables and sliders. I should probably make it in Unity.
  • DSlides: A Powerpoint inspired presentation maker. The UI is powered by pygameUI. Even has its own presentation file language, to easly edit it without the editor. Of course it's not even close to Powerpoint complexity, But still a nice lightweight tool.
  • Clicky: A super simple and stupid autoclicker. It's powered by DamGUI tho! And that's the only reason why it exist, i wanted to test the library.
  • DamPad: Very simple note and book editor made using customtkinter. Really I was just testing the UI framework, but it ended up a neat program.

Simulators

  • DamusOS: A pygame simulator that uses pygame_gui for UI that tries to simulate an Operating System. Has an API that let user add custom programs.
